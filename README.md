# safeself-bench

A synthetic benchmark for studying what an LLM can infer about a person from
their digital footprint.

This repository releases:

- **48 persona specs** across 16 adversary-style archetypes × 3 target
  instantiations each. Every spec is laid out on a 32-dimension schema
  spanning identity, device/OPSEC, daily rhythm, vulnerability, and relations.
- **111 synthetic activity bundles** organised into 7 *series*. Each bundle
  represents 30 days of multimodal digital activity (searches, messages,
  calendar, videos, media, locations) for one persona.
- **A 48-question fidelity bank** and **48 answer keys** for ground-truth
  scoring against the personas.
- **Scorers** for fidelity (an LLM judge answers the questions from each
  bundle) and realism (an LLM judge rates how plausibly human each bundle
  reads).
- **A text-only filter** that converts multimodal bundles into a text-only
  view, useful for adversary models that ingest text only.

## Why this exists

When a user hands a third party their digital footprint — a Google Takeout, a
Facebook archive, or any equivalent personal-data export — a current-generation
LLM can recover a startling amount about that person: not just the obvious
fields, but rhythm, relationships, and vulnerabilities.

Studying that inference problem rigorously needs synthetic data. Real human
data is hard to come by at scale, and analysing it without explicit consent is
not something this work is willing to do. This benchmark gives the field a
reproducible substrate — same personas, same questions, same scorers —
so different research groups can compare results.

## What is *not* released

- **Real user data of any kind.** Everything here is synthetic.

## Repository layout

```
personas/
  schema/           32-dimension persona schema (modules + dimensions)
  specs/            48 persona specs (16 archetypes × target_1/2/3)
cohorts/
  series_a/         16 bundles · Gemma 4              (target_1, dejittered)
  series_b/         16 bundles · Gemma 4              (target_2, dejittered)
  series_c/         16 bundles · Llama 3.3 70B        (target_3, jittered only)
  series_d/         16 bundles · gpt-oss 120B         (target_1, dejittered)
  series_e/         16 bundles · gpt-oss 120B         (target_2, dejittered)
  series_f/         16 bundles · Claude Opus 4.X      (target_2, dejittered)
  series_g/         15 bundles · GPT-5.4 / Codex      (target_1, dejittered)
answer_keys/        48 ground-truth answer files
question_bank.json  48-question fidelity bank
scoring/
  fidelity_evaluate.py    score one bundle against its answer key
  fidelity_sweep.py       score a whole cohort with one or more LLM judges
  fidelity_semantic.py    semantic-similarity operator used by the scorer
  realism_judge.py        rate how plausibly human a bundle reads
  textonly_filter.py      derive a text-only view of any bundle
```

## How the cohorts were made

Each bundle was generated by an LLM agent taking a persona spec as input and
emitting 30 days of multimodal digital-footprint activity. Five different LLM
agents were used across the seven series, showcasing how the same persona
archetypes can produce diverse but coherent footprints under different
generators. Generation is non-deterministic — exact bundles cannot be
reproduced from the persona spec alone.

The **jittered / dejittered** distinction refers to timestamp and coordinate
noise: jittered bundles preserve fine-grained noise that simulates raw export
data; dejittered bundles snap timestamps and coordinates to cleaner values for
easier downstream analysis. Series C is jittered-only (no dejittered version
was produced for that generator).

## Usage

### Score a single bundle for fidelity

```bash
python3 scoring/fidelity_evaluate.py \
    --bundle cohorts/series_a/the_automated_phisher_target_1_intake.json \
    --answer-key answer_keys/the_automated_phisher_target_1.json \
    --question-bank question_bank.json \
    --judge opus \
    --out report.json
```

### Sweep a whole cohort with a panel of judges

```bash
python3 scoring/fidelity_sweep.py \
    --bundle-dir cohorts/series_a \
    --answer-keys-dir answer_keys \
    --question-bank question_bank.json \
    --judges opus,codex,gemini \
    --out-dir scores/series_a
```

### Generate a text-only view of a cohort

```bash
python3 scoring/textonly_filter.py --cohort cohorts/series_a/ --out series_a_textonly/
```
